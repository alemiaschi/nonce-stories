# The Brolm — Nonce Stories

A literary-interactive web app. Readers explore a tree of short stories composed entirely of invented (*nonce*) words. Clicking a nonce word navigates into a sub-story that contextually reveals its meaning. The deeper you go, the more intelligible the language becomes.

**Live site:** [alemiaschi.github.io/nonce-stories](https://alemiaschi.github.io/nonce-stories/)

---

## What is this?

Every content word in the root story — every noun, verb, adjective, adverb — is a nonce word: a pure sound with a grammatical role but no assigned meaning. Only the grammatical skeleton is English.

Each nonce word is a portal. Click on one and you descend into a sub-story that places that word in new situations — not defining it, but illuminating it the way a child acquires vocabulary: through context, repetition, and inference.

The project grows over time. Each iteration expands one frontier word into a new sub-story. Meaning is never pre-planned — it is *discovered in the act of writing*, from the syntactic and narrative context in which the word first appeared.

This is a collaborative creative project, conceived and written in conversation with [Claude](https://claude.ai) (Anthropic). The web application was built with [Claude Code](https://claude.ai/code).

---

## Data

The raw data lives in `data/nonce_stories_memory.json` — a structured JSON file containing the full lexicon, all story texts, and the expansion log. If you want to work with the data directly, that is the file to read.

`data/stories_data.json` is a derived, frontend-optimised version generated by the build script.

---

## Running locally

Requires Node.js 20+ and Python 3.

```bash
npm install
python3 scripts/build_stories_data.py
npm run dev
```

The app runs at `http://localhost:5173/nonce-stories/`.

To regenerate the frontend data after updating the memory file:

```bash
python3 scripts/build_stories_data.py
```
